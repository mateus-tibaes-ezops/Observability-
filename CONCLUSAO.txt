📊 ALERTAS CONFIGURADOS

  🔴 CRÍTICOS (Resposta Imediata)
     1. ServiceDown           → Aplicação offline por 30s
     2. HighCpuCritical       → CPU > 95% por 1 minuto
     3. LowMemoryAvailable    → Memória < 10% por 2 minutos
     4. PrometheusDown        → Monitoramento offline

  🟡 WARNINGS (Investigar em 10 min)
     5. HighCpuWarning        → CPU > 80% por 2 minutos
     6. HighErrorRate         → Erros 5xx > 5% por 5 minutos
     7. HighLatencyP99        → Latência > 1s por 5 minutos

  Cada alerta com:
  ✓ Severity label
  ✓ Runbook URL
  ✓ Dashboard URL
  ✓ Descrição detalhada
  ✓ Instruções de resposta

❓ PERGUNTAS FREQUENTES

  P: Como alterar o webhook do Slack?
  R: Editar .env → SLACK_WEBHOOK_URL → docker compose restart alertmanager

  P: Como adicionar novo alerta?
  R: Editar prometheus/alerts.rules.yml → adicionar regra → restart prometheus

  P: Como customizar threshold de CPU?
  R: Editar prometheus/alerts.rules.yml → expr: ... > 90 (mudar de 80)

  P: Posso rodar em produção agora?
  R: Sim! Com considerações: TLS, autenticação, backups, load balancer

  P: E se não quiser Slack?
  R: Alertmanager funciona sem Slack. Acessar UI em http://localhost:9093
